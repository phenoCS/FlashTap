一、项目目标
============================================================

项目名称：FlashTap 一键本地AI编程环境部署
平台：Windows 10/11
目标：双击一个脚本，全自动完成——
  1. 安装/配置 Ollama 本地大模型引擎
  2. 安装 VS Code + Continue AI 插件 + 中文语言包
  3. 下载 Qwen2.5-Coder 7B 模型并导入 Ollama
  4. 配置 WSL Ubuntu + g++ + C++ 调试环境
  5. 自动打开 VS Code，侧边栏 AI 即可用
要求：全程零手动、零命令行、零弹窗

三、已解决的关键问题（2026-06-17 修复，勿再改动）
============================================================

🔧 问题 A：Continue 插件 "No models configured"（已解决）
--------------------------------------------------------------
  根因：
    Continue 插件更新后，配置格式从显式指定模型名改为 AUTODETECT
    自动检测。旧格式使用 "model: qwen2.5-coder:7b" 不再被识别。

  修复（涉及文件：config.yaml / config.json / config.ts / configure-continue.py）：
    - config.yaml: 改为 schema v1 + AUTODETECT 格式
      models:
        - name: Autodetect
          provider: ollama
          model: AUTODETECT
    - config.json: 同步改为 AUTODETECT
    - config.ts: 同步改为 AUTODETECT
    - configure-continue.py: CONFIG_YAML/CONFIG_JSON/config.ts 生成
      全部改为 AUTODETECT，验证逻辑同步更新

  ⚠️ 不要改回旧的显式模型名格式！

🔧 问题 B：双窗口问题（已解决）
--------------------------------------------------------------
  根因：
    Setup-FlashTap.ps1 中，configure-continue 完成后先执行了
    code --command "workbench.action.reloadWindow"（触发第一个窗口），
    然后又在最后 kill 所有 VS Code 进程并重启（触发第二个窗口）。

  修复（涉及文件：Setup-FlashTap.ps1）：
    - 删除了冗余的 reloadWindow 步骤
    - 只保留最后的 kill + 重启，确保只有一个窗口

🔧 问题 C：杀进程误杀其他用户的 VS Code（已解决）
--------------------------------------------------------------
  根因：
    Get-Process Code | Stop-Process 会杀掉所有用户的 Code.exe 进程。

  修复（涉及文件：Setup-FlashTap.ps1）：
    - 改为 taskkill /F /FI "USERNAME eq $env:USERNAME" /IM Code.exe
    - 只杀当前用户的 VS Code 进程，不影响其他用户

🔧 问题 D：configure-continue.py UnicodeEncodeError 崩溃（已解决）
--------------------------------------------------------------
  根因：
    Windows 控制台 GBK 编码无法打印 ✓ (U+2713) 字符。

  修复（涉及文件：configure-continue.py / download-models.py）：
    - ✓ → [OK]
    - print() 包裹 try/except UnicodeEncodeError，回退为 ASCII

🔧 问题 E：PS1 文件双重 BOM 导致语法错误（已解决）
--------------------------------------------------------------
  根因：
    install-localcoder.ps1 / install-vscode.ps1 / setup-cpp-env.ps1
    文件头有双重 UTF-8 BOM (ef bb bf ef bb bf)，第二个 BOM 被
    PowerShell 解析为命令名 ﻿ 导致报错。

  修复：
    - 去掉多余 BOM，只保留一个

🔧 问题 F：中文用户名路径导致 Ollama 模型加载失败（已解决）
--------------------------------------------------------------
  根因：
    Ollama Windows 服务以 SYSTEM 账户运行，看不到用户级环境变量
    OLLAMA_MODELS=D:\ollama_models，默认使用 C:\Users\本人\.ollama\
    （中文路径），llama.cpp 无法处理中文路径。

  修复（涉及文件：download-models.py）：
    - 部署模型前停止 Windows 服务
    - 设置 OLLAMA_MODELS + OLLAMA_HOME 到 D 盘
    - 通过 PowerShell Start-Process 启动 ollama serve（无黑窗）
    - 模型文件存储到 D:\ollama_models（无中文路径）

🔧 问题 G：PowerShell Run-Python 输出被吞掉（已解决）
--------------------------------------------------------------
  根因：
    $result = & python ... 2>&1 把所有输出捕获到变量，控制台不可见。

  修复（涉及文件：Setup-FlashTap.ps1）：
    - 改为 & python ...（直接输出，不捕获）

四、待解决的问题
============================================================

🔴 问题 1（唯一剩余）：WSL 内 C/C++ 调试器扩展安装失败
--------------------------------------------------------------
  现象：
    - [警告] 扩展安装失败，未检测到 cpptools 目录
    - 调试输出：UNZIP_CPP_FAIL 或 zipfile.BadZipFile
    - 用户反馈：打开 VS Code 后 C++ 编译报"与环境不匹配"

  根因：
    - 旧方案：curl 从 VS Code Marketplace API 下载 .vsix 再解压
    - Marketplace API 返回的不是有效 zip 文件（可能是 HTML 重定向页）
    - 导致 zipfile.BadZipFile: File is not a zip file

  最新修复（2026-06-17）：
    - 放弃 curl 下载 .vsix 的方式
    - 改用 VS Code Server 自带的 CLI 安装扩展：
      $HOME/.vscode-server/bin/{commitId}/bin/remote-cli/code --install-extension
    - 这是 VS Code 原生安装扩展的方式，100% 可靠

  涉及文件：Setup-FlashTap.ps1（extScript 部分）
